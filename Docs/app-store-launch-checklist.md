## 二、资质与合规要求（关键，必须先做）

### 2.1 软件著作权（软著）

**所有国内应用商店强制要求。**

- [ ] 申请《计算机软件著作权登记证书》
  - 在中国版权保护中心申请（https://register.ccopyright.com.cn）
  - 准备材料：源代码前 30 页 + 后 30 页、软件说明书、申请表
  - 周期：普通 30 个工作日，加急可至 3-5 个工作日
  - 费用：普通件免费，加急需付费

### 2.2 ICP 备案（应用所配后端域名）

**有后端服务的应用必须提供 ICP 备案号。**

- [ ] 确认 ChildNotes 后端所用域名已通过 ICP 备案
- [ ] 在应用商店上架材料中填写备案号
- [ ] 若使用云服务器（阿里云/腾讯云），备案需服务器在境内

### 2.3 应用备案（工信部新规，2024 年起强制）

**依据《工业和信息化部 国家互联网信息办公室关于开展移动互联网应用程序备案工作的通知》。**

- [ ] 在应用分发平台完成 APP 备案
- [ ] 提供主办单位信息、APP 名称、包名、签名 MD5 等
- [ ] 备案通过后方可上架（未备案应用商店有权拒绝上架）

### 2.4 特殊资质（视功能而定）

ChildNotes 涉及儿童信息记录，**不属于医疗类应用**（不提供医疗建议/诊断），但需注意：

- [ ] 确认 App 不涉及"医疗器械""医疗诊断"等敏感词，避免触发医疗资质审核
- [ ] 若有 AI 分析功能（已有 [AiAnalysisView](file:///e:/0_Code/5_Git/AiJi/ChildNotes/ChildNotes/Views/AiAnalysisView.axaml)），需在应用描述中明确标注"AI 建议仅供参考，不构成医疗诊断"
- [ ] 如涉及生成式 AI 服务（深度合成），可能需要《深度合成服务算法备案》

### 2.5 公司 / 个人主体资质

- [ ] 营业执照（企业开发者）或身份证（个人开发者）
- [ ] 部分商店（华为、小米）个人开发者上架应用限制较多，建议注册企业主体

- [ ] 准备 **1024×1024 px** 备用图

- [ ] 截图建议加简洁说明文字（不要过度营销）

### 3.4 宣传视频（可选）

## 四、隐私合规（重点，国内审核最严格项）

### 4.1 隐私政策

- [ ] 撰写正式版《隐私政策》完整文本（不仅是 App 内摘要）
- [ ] 在公司官网 / 后端服务部署可公网访问的隐私政策页面（URL）
- [ ] 内容必须包含：
  - 信息收集清单（设备信息、日志、账号、宝宝信息等）
  - 信息使用目的
  - 信息共享、转让、披露规则
  - 信息存储与保护措施
  - 用户权利（查询、更正、删除、注销）
  - 未成年人保护条款（**ChildNotes 涉及儿童信息，此项必备**）
  - 联系方式
- [ ] App 内隐私协议弹窗：当前 [PrivacyConsentView](file:///e:/0_Code/5_Git/AiJi/ChildNotes/ChildNotes/Views/PrivacyConsentView.axaml) 已实现，但需确认摘要文本是否与最终版政策一致

### 4.2 用户协议

- [ ] 撰写《用户服务协议》
- [ ] App 内首次启动需同时展示隐私政策 + 用户协议

### 4.3 权限说明

当前 [AndroidManifest.xml](file:///e:/0_Code/5_Git/AiJi/ChildNotes/ChildNotes.Android/Properties/AndroidManifest.xml) 仅申请 `INTERNET` 权限，**非常干净**。但仍需：

- [ ] 在隐私政策中逐项列明权限及用途
- [ ] 代码层验证：运行时无任何动态申请权限（如 storage / location / camera 等）
- [ ] 若后续增加权限，必须先更新隐私政策并同步到商店

### 4.4 SDK 清单（合规要求）

商店要求披露所有第三方 SDK：

- [ ] 整理 SDK 清单表格：
  - SDK 名称、所属公司、用途、收集信息、隐私政策链接
- [ ] 当前已知依赖：
  - Avalonia.Android（UI 框架）
  - Microsoft.Data.Sqlite + SQLitePCLRaw.bundle_e_sqlite3（本地数据库）
  - Microsoft.Extensions.* 系列
  - CommunityToolkit.Mvvm
- [ ] 检查是否集成任何"广告 SDK""统计 SDK""推送 SDK"（当前看起来没有，但需确认 CI 产物不包含）

### 4.5 未成年人保护

ChildNotes 记录儿童信息，必须额外处理：

- [ ] 隐私政策中单独章节说明儿童信息处理规则
- [ ] 用户注册时明示"仅限家长/监护人使用"
- [ ] 不收集儿童本人联系方式
- [ ] 不做儿童画像分析用于商业推送

## 五、应用功能与质量

### 5.1 应用内容合规

- [ ] 确认所有文案不含以下违规内容：
  - 医疗诊断承诺（避免被判定为医疗 App）
  - 涉政、涉黄、涉赌、涉暴
  - 诱导分享 / 刷屏营销
  - 未经授权的第三方商标 / IP
- [ ] AI 分析功能的输出文案需加免责声明

### 5.2 目标 SDK 版本

- [ ] 各商店最低要求 targetSdk 一般为 **34（Android 14）或 35（Android 15）**
- [ ] 当前 csproj 未显式设置 `TargetFrameworkVersion`，需确认 .NET 10 Android SDK 默认值
- [ ] 当前 `minSdk` 通过 `SupportedOSPlatformVersion=24` 设定为 Android 7.0，符合主流商店要求

### 5.3 ABI 与包体

- [ ] 当前 CI 只产出 APK（非 AAB）
  - 国内商店基本接受 APK 格式
  - Google Play 已强制 AAB，但国内商店不强制
- [ ] 确认 APK 包含 `arm64-v8a`（必须）+ `armeabi-v7a`（兼容老机型）+ `x86_64`（模拟器测试）
  - 见 obj 目录：`lp/99/jl/jni/` 已含全部 ABI 的 `libe_sqlite3.so`
- [ ] 包体建议控制在 **50MB 以内**（Avalonia + .NET 运行时通常 30-40MB）
  - 超过 100MB 部分商店会限制 Wi-Fi 下载或提示

### 5.4 64 位支持

- [ ] 确保所有 `.so` 库均有 arm64-v8a 版本（Google Play 强制，国内商店建议）

### 5.5 签名一致性

- [ ] **不同商店必须使用同一个 keystore**，否则无法覆盖升级
- [ ] 当前 CI 已通过 GitHub Secrets 固定签名 ✅
- [ ] keystore 文件需**离线备份**（丢失则无法更新应用）

## 七、上架前技术检查清单

### 7.1 包名确认

- [ ] 当前包名 `com.CompanyName.ChildNotes` 为占位符，**发布前必须改为正式包名**
- [ ] 改包名后需要：重新生成 keystore、所有商店材料统一、旧版用户无法覆盖升级（首次发布无所谓）

### 7.2 版本号

- [ ] 首次上架建议 `v1.0.0`（**不要用 0.x 版本号上架商店**，用户感知不成熟）
- [ ] 确认 [release.yml](file:///e:/0_Code/5_Git/AiJi/.github/workflows/release.yml) 的版本号解析逻辑正确
- [ ] versionCode 必须为正整数且每次更新递增

### 7.3 Release 构建配置

- [ ] 确认 `RunAOTCompilation=false` 在 Release 下生效（避免 XA1030）
- [ ] 确认 `PublishTrimmed=false`（未启用裁剪）
- [ ] 确认 `AndroidEnableProguard=false`（未启用代码混淆）
  - **注意**：部分商店要求启用混淆以通过安全检测，需评估
- [ ] 确认 `SourceRevisionId` 为空（versionName 无 git 后缀）

### 7.4 无障碍

- [ ] 当前 [MainActivity.cs](file:///e:/0_Code/5_Git/AiJi/ChildNotes/ChildNotes.Android/MainActivity.cs) 中 `DisableAvaloniaAccessibility()` 主动禁用了 TalkBack
  - 商店审核可能要求无障碍支持（**风险点**）
  - 需在审核材料中说明"Avalonia 框架 bug 临时规避，后续版本修复"

### 7.5 崩溃与日志

- [ ] 确认 Release 构建中无 `Console.WriteLine` / `Debug.WriteLine` 残留
- [ ] 确认日志级别在 Release 下降为 Warning/Error
- [ ] 建议接入崩溃收集（Bugly / 友盟 等国内服务）以便线上问题定位

### 7.6 网络请求

- [ ] 所有 HTTP 请求必须使用 HTTPS（部分商店审核会拦截 HTTP 请求）
- [ ] 后端 API 域名需 ICP 备案

### 7.7 权限最小化

- [ ] 确认 `AndroidManifest.xml` 仅申请必要权限
- [ ] 不要申请 `READ_EXTERNAL_STORAGE` / `WRITE_EXTERNAL_STORAGE`（使用 Scoped Storage）
- [ ] 不要申请 `ACCESS_FINE_LOCATION`（与育儿记录无关）
