# 快捷记录改造：悬浮 + 按钮 + 扇形菜单 + 居中卡片表单

## 摘要

将首页底部"快捷记录"常驻白色面板（带凸型拉手 + 9 宫格按钮）替换为**屏幕右下角的悬浮 + 按钮**：点击后菜单项围绕 + 按钮呈**扇形向外展开**；选中某项后弹出**居中卡片表单**（不再是底部抽屉），避免输入法遮挡。原有的 `RecordSheetView`（底部抽屉）保留不动，继续供疫苗补记、活动记录等其他入口使用。

### 1. 新建 `QuickMenuViewModel.cs`（扇形菜单 + 卡片表单的状态机）

**文件**：`ChildNotes\ChildNotes\ViewModels\QuickMenuViewModel.cs`（新建）

职责：管理 "+ 按钮" 的菜单展开/收起状态，以及"卡片表单"的显示状态。复用 `RecordSheetViewModel` 的表单逻辑（直接组合，不重写）。

```csharp
public partial class QuickMenuViewModel : ViewModelBase
{
    [ObservableProperty] private bool _isMenuOpen;       // 扇形菜单是否展开
    [ObservableProperty] private bool _isCardOpen;       // 卡片表单是否显示
    [ObservableProperty] private string _cardTitle = string.Empty;

    // 复用 RecordSheetViewModel 的表单 + 保存逻辑
    public RecordSheetViewModel RecordSheet { get; }
    public ObservableCollection<QuickActionItem> Actions { get; }  // 9 个快捷项

    public event Action? Saved;  // 保存后通知 MainShell 刷新首页

    public QuickMenuViewModel(RecordSheetViewModel recordSheet)
    {
        RecordSheet = recordSheet;
        RecordSheet.Saved += () => { IsCardOpen = false; Saved?.Invoke(); };
        // 9 个快捷项（从 HomeViewModel 迁移，去掉 Sleep 用 🌡️/💊/🍶/🥣/🍽️/📏 等）
        Actions.Add(new QuickActionItem("🍼", "喂奶", RecordType.Feed, "#FFF0E6"));
        // ... 共 9 项
    }

    [RelayCommand] private void ToggleMenu() => IsMenuOpen = !IsMenuOpen;
    [RelayCommand] private void CloseMenu() => IsMenuOpen = false;

    [RelayCommand]
    private void Select(string type)
    {
        IsMenuOpen = false;
        RecordSheet.Open(type);   // 复用现有 Open 方法
        IsCardOpen = true;
        CardTitle = RecordSheet.Title;
    }

    [RelayCommand] private void CloseCard() { IsCardOpen = false; RecordSheet.Close(); }
}
```

**决策说明**：复用 `RecordSheetViewModel` 实例（与 MainShell 现有的 `_recordSheet` 是同一实例），避免双份状态。`Open()` 方法已设置 `IsVisible=true`，但卡片 View 的显隐用 `IsCardOpen` 控制，`RecordSheet.IsVisible` 维持 true 即可。

### 2. 新建 `QuickMenuView.axaml`（悬浮 + 按钮 + 扇形菜单 + 卡片容器）

**文件**：`ChildNotes\ChildNotes\Views\QuickMenuView.axaml`（新建）+ `.cs`

结构（顶层 `Panel` 透明，覆盖在 MainShell 之上但仅在命中 + 按钮或菜单/卡片打开时拦截事件）：

```
Panel (IsHitTestVisible 仅在 IsMenuOpen||IsCardOpen 时为 true 的全屏遮罩层；+ 按钮始终可点)
├── Border (遮罩，#80000000，IsVisible=IsMenuOpen||IsCardOpen，点击=关闭菜单/卡片)
├── 扇形菜单 (IsVisible=IsMenuOpen)
│   └── ItemsControl 绑定 Actions，每个菜单项用 Canvas 定位到扇形位置
│       └── Border (圆形 48x48, Background=IconBg) + TextBlock(emoji) + TextBlock(label)
├── 卡片表单 (IsVisible=IsCardOpen)
│   └── Border 居中 (CornerRadius=16, MaxWidth=320, MaxHeight=500, Background=白)
│       └── Grid (标题 + Close ✕ + ContentControl Content={Binding RecordSheet} 嵌入 RecordSheetView 的表单部分)
└── + 悬浮按钮 (右下角，VerticalAlignment=Bottom Margin=0,0,16,72 避开 TabBar)
    └── Button (圆形 56x56, Background=WeuiGreenBrush, BoxShadow)
        └── TextBlock "+" (旋转 45° 当 IsMenuOpen 时变为 "×")
```

**扇形定位实现**（关键）：
- 9 项分布在 180°→90° 的左上象限扇形（+ 按钮在右下角，菜单向左上展开）
- 角度：从 180°（正左）到 90°（正上），等分 8 段 → 9 项角度为 `180° - i * (90°/8)`
- 半径：80px（每项中心距 + 按钮中心）
- 每项位置：`x = -radius * cos(angle)`，`y = -radius * sin(angle)`（相对 + 按钮中心）

**两种实现选项**（采用 A）：

- **选项 A（推荐）**：在 ViewModel 中为每个 `QuickActionItem` 计算好 `OffsetX`/`OffsetY`，XAML 中用 `RenderTransform` 绑定。简单、无动画。
- **选项 B**：用 `Canvas` + 代码后台手动定位，加 `Transitions` 做展开动画。复杂但效果更好。

**卡片表单内容复用**：
- 卡片内的 `ContentControl Content="{Binding RecordSheet}"` 会通过 `ViewLocator` 自动映射到 `RecordSheetView`
- 但 `RecordSheetView` 自带遮罩 + 底部抽屉定位，直接嵌入会冲突
- **解决方案**：新建 `QuickRecordCardView.axaml`（精简版），只保留 `RecordSheetView` 中间的表单 `ScrollViewer` 部分（去掉外层遮罩 + 底部对齐），通过 `ContentControl` 或直接复制表单 XAML 实现
- **简化方案（采用）**：`QuickRecordCardView` 内部直接引用 `RecordSheetViewModel` 的各子 Form（`FeedForm`/`DiaperForm` 等），复用 [RecordSheetView.axaml:169-803](file:///e:/0_Code/5_Git/AiJi/ChildNotes/ChildNotes/Views/RecordSheetView.axaml#L169-L803) 的表单 XAML 段（剔除疫苗/活动/异常这 3 个非快捷项，只保留 Feed/Diaper/Sleep/Temperature/Growth/Supplement/Pump/Complementary/MaternalFood 共 9 项）

## 假设与决策

| # | 决策 | 理由 |
|---|------|------|
| 1 | 扇形菜单采用 ViewModel 计算 `OffsetX/OffsetY` + `RenderTransform` 绑定，**不加展开动画** | 简单可靠；动画可作为后续增强 |
| 2 | 卡片表单新建独立 View（`QuickRecordCardView`），不复用 `RecordSheetView` | `RecordSheetView` 是底部抽屉结构（遮罩+`VerticalAlignment=Bottom`），无法直接居中；新建可精准控制卡片样式 |
| 3 | 卡片表单只支持 9 种快捷类型（Feed/Diaper/Sleep/Temp/Growth/Supp/Pump/Comp/MaternalFood），不含 Vaccine/Activity/Abnormal | 这 3 种入口仍走原 `RecordSheetView` 底部抽屉（疫苗时间轴太复杂，不适合卡片） |
| 4 | 悬浮 + 按钮始终可见（在所有 tab），不限于首页 | 提升全局快捷记录体验；按钮定位避开 TabBar |
| 5 | 复用 `RecordSheetViewModel` 单一实例 | 表单状态/保存逻辑已封装好，避免双份 |
| 6 | `QuickActionItem` 类保留在 `HomeViewModel.cs` | 避免新建文件；该类是纯数据类，无副作用 |
| 7 | 扇形角度范围 180°→90°（左上象限），9 项等分 8 段 | + 按钮在右下角，菜单向左上展开最自然，不遮挡 TabBar |
| 8 | 遮罩点击/卡片 ✕ 关闭均调用 `CloseMenu`/`CloseCard` 命令 | 保持 ViewModel 驱动，符合 MVVM |
