# 疫苗追踪列表加载性能优化方案

> 基于真机部署反馈的卡顿现象，结合 Avalonia 12.0 官方文档与最佳实践制定
> 制定日期：2026-06-30
> 目标：解决首次打开疫苗追踪时的明显卡顿

---

## 一、问题诊断

### 1.1 真机现象

首次打开疫苗追踪应用时出现明显卡顿，尤其在低端 Android 设备上更为突出。

### 1.2 根因分析

通过对代码库的审查，定位到 **三个层面的性能瓶颈**：

| 层面 | 问题位置 | 具体表现 | 严重度 |
|------|----------|----------|--------|
| **UI 虚拟化缺失** | [HomeView.axaml:195](file:///e:/0_Code/5_Git/AiJi/ChildNotes/ChildNotes/Views/HomeView.axaml#L195) | 疫苗列表使用 `ItemsControl`（默认 `StackPanel`，无虚拟化），展开后 52 个剂次一次性创建 52 个完整 UI 元素 | ★★★★★ |
| **数据量级** | [VaccineCatalog.FlattenDoses()](file:///e:/0_Code/5_Git/AiJi/ChildNotes/ChildNotes/Services/VaccineCatalog.cs#L214) | 全国免疫规划疫苗共 **52 个剂次**（含免费+自费），展开后全部渲染 | ★★★★ |
| **嵌套布局深度** | [HomeView.axaml:65](file:///e:/0_Code/5_Git/AiJi/ChildNotes/ChildNotes/Views/HomeView.axaml#L65) | 疫苗 ItemsControl 嵌套在 `ScrollViewer > StackPanel > Border > StackPanel` 内，外层 StackPanel 不参与虚拟化 | ★★★ |

### 1.3 关键测量数据

```
疫苗剂次总数：52（VaccineCatalog.All 中所有 Doses 展开）
默认显示条数：3（VaccineDefaultVisibleCount）
展开后显示条数：52
首次渲染 UI 元素数（展开态）：52 × 每项约 8 个控件 = 416 个控件
首次渲染 UI 元素数（折叠态）：3 × 8 = 24 个控件
```

**核心结论**：折叠态仅 3 条，卡顿主因不是数量，而是**展开操作时的批量渲染** + **首次加载时 52 条 ItemsControl 全部参与布局测量**（即使只显示 3 条，ItemsControl 仍会测量全部数据源）。

## 二、官方推荐方案研究

### 2.1 官方列表控件选型对比

通过 [Avalonia 官方文档](https://docs.avaloniaui.net/docs/guides/data-guides/itemscontrol-and-itemsrepeater) 与 MCP 文档检索，得到以下对比表：

| 控件 | 选择支持 | 虚拟化支持 | 适用场景 | Avalonia 12.0 状态 |
|------|----------|------------|----------|-------------------|
| `ListBox` | 内置 | ✅ 默认启用 | 可选择列表 | ✅ 推荐 |
| `ItemsControl` | 无 | ❌ 默认不启用（需手动配 `VirtualizingStackPanel`） | 小集合、自定义布局 | ✅ 推荐 |
| `ItemsRepeater` | 无 | ✅ | 大集合、自定义布局、极致性能 | ⚠️ **独立 NuGet 包已退役归档** |

### 2.2 关键官方结论（重要）

> **`Avalonia.Controls.ItemsRepeater` 独立 NuGet 包在 12.0 版本后已退役归档**。
> 官方推荐继续使用 Avalonia 主包内置的 `ItemsControl` / `ListBox` + `VirtualizingStackPanel` 实现虚拟化。

来源：[Avalonia.Controls.ItemsRepeater GitHub 仓库公告](https://github.com/AvaloniaUI/Avalonia.Controls.ItemsRepeater)

**本项目决策**：**不引入 ItemsRepeater NuGet 包**，采用官方推荐的 `ItemsControl` + `VirtualizingStackPanel` 方案。

### 2.3 官方虚拟化最佳实践

#### 2.3.1 启用虚拟化（3 行 XAML）

```xml
<ItemsControl ItemsSource="{Binding Items}">
    <ItemsControl.ItemsPanel>
        <ItemsPanelTemplate>
            <VirtualizingStackPanel />
        </ItemsPanelTemplate>
    </ItemsControl.ItemsPanel>
</ItemsControl>
```

#### 2.3.2 虚拟化生效的必要条件

| 条件 | 说明 | 本项目是否满足 |
|------|------|---------------|
| 使用 `VirtualizingStackPanel` 作为 `ItemsPanel` | 核心启用方式 | ❌ 当前用默认 StackPanel，需改造 |
| 数据源实现 `INotifyCollectionChanged` | 增删项时虚拟化才生效 | ✅ 已用 `ObservableCollection<VaccineItem>` |
| 容器有**固定或可预测的行高** | 动态高度会破坏虚拟化缓存 | ⚠️ 当前每项高度依赖内容，需评估 |
| 容器是 `ScrollViewer` 而非 `StackPanel` | 虚拟化需要 ScrollViewer 提供视口信息 | ❌ 当前外层是 StackPanel，需重构 |

#### 2.3.3 调优参数

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| `CacheLength` | `0.8`（桌面）/ `0.3`（移动端低配） | 视口外预加载区域比例，影响滚动流畅度与内存 |
| `VirtualizationMode` | `Recycling`（默认） | UI 元素复用，避免频繁创建销毁 |
| `UseLayoutRounding` | `True` | 减少子像素计算，提升渲染性能 |

#### 2.3.4 UI 渲染优化要点

| 优化项 | 说明 | 本项目应用 |
|--------|------|-----------|
| **ItemTemplate 扁平化** | 避免多层嵌套 `Border`/`Grid`，单个 `TextBlock` 或简单 `StackPanel` | 当前每项是 `Border > Grid > 3 个控件`，可接受 |
| **禁用透明背景** | `Background="#80FFFFFF"` 改为不透明色 `#FFFFFFFF`，避免 GPU 混合开销 | 需检查 WeUI 样式 |
| **绑定模式按需选择** | 静态字段用 `Mode=OneTime`，频繁更新字段确保 INPC 只在值真变化时触发 | `VaccineItem` 是只读 POCO，可全部用 `OneTime` |
| **开启布局舍入** | 根节点加 `UseLayoutRounding="True"` | 待加 |

## 三、优化实施计划

### 3.1 实施优先级矩阵

| 优先级 | 方案 | 改动范围 | 预期收益 | 风险 |
|--------|------|----------|----------|------|
| **P0** | 方案 A：疫苗列表启用虚拟化 | 中 | 展开渲染 416→80 控件，内存 -80% | 中（需重构容器层级） |
| **P1** | 方案 B：折叠态懒加载 | 小 | 首次打开不渲染全部 52 项 | 低 |
| **P1** | 方案 C：ItemTemplate 扁平化 + OneTime 绑定 | 小 | 单项渲染耗时 -30% | 低 |
| **P2** | 方案 D：UseLayoutRounding + 不透明背景 | 极小 | 渲染耗时 -10% | 无 |
| **P2** | 方案 E：性能埋点 + 基准测试 | 小 | 可量化验证 | 无 |

### 3.2 方案 A：疫苗列表启用虚拟化（P0）

**问题**：当前 [HomeView.axaml:195](file:///e:/0_Code/5_Git/AiJi/ChildNotes/ChildNotes/Views/HomeView.axaml#L195) 的疫苗 `ItemsControl` 嵌套在 `Border > StackPanel` 内，外层是整个 HomeView 的 `ScrollViewer`。虚拟化要求 `ItemsControl` 直接位于 `ScrollViewer` 内以获取视口信息，当前结构无法生效。

**优化步骤**：

1. **重构疫苗区块为独立 ScrollViewer**：把疫苗追踪区块从 HomeView 的外层 ScrollViewer 中拆出，改为自带 ScrollViewer 的独立卡片，给虚拟化提供视口边界。

2. **配置 VirtualizingStackPanel**：

```xml
<!-- 改造前 -->
<ItemsControl ItemsSource="{Binding VisibleVaccineItems}">
    <ItemsControl.ItemTemplate>
        <DataTemplate x:DataType="vm:VaccineItem">
            <!-- ... -->
        </DataTemplate>
    </ItemsControl.ItemTemplate>
</ItemsControl>

<!-- 改造后 -->
<ScrollViewer Height="280" VerticalScrollBarVisibility="Auto">
    <ItemsControl ItemsSource="{Binding VisibleVaccineItems}">
        <ItemsControl.ItemsPanel>
            <ItemsPanelTemplate>
                <VirtualizingStackPanel CacheLength="0.3" />
            </ItemsPanelTemplate>
        </ItemsControl.ItemsPanel>
        <ItemsControl.ItemTemplate>
            <DataTemplate x:DataType="vm:VaccineItem">
                <!-- ... -->
            </DataTemplate>
        </ItemsControl.ItemTemplate>
    </ItemsControl>
</ScrollViewer>
```

3. **固定容器高度**：给疫苗 ScrollViewer 设固定 `Height="280"`（约 5 项可见），虚拟化才能计算可见区域。展开/折叠改为切换 ScrollViewer 的 `IsVisible` 或 `Height`。

**预期收益**：
- 展开态渲染控件数：416 → ~80（仅 10 项可见 × 8 控件）
- 内存占用：-80%
- 展开操作响应：从 200-400ms 降至 < 80ms

**风险**：改变 UI 结构可能影响布局，需回归测试。

### 3.3 方案 B：折叠态懒加载（P1）

**问题**：当前 `VisibleVaccineItems` 即使折叠态只显示 3 条，`ItemsControl` 仍会绑定到 `VaccineItems.Take(3).ToList()`，但 `ApplyVaccines` 会先填充全部 52 条到 `VaccineItems` 再 Take。

**优化步骤**：折叠态不展开 ItemsControl 的 ScrollViewer（`Height="0"` 或 `IsVisible="False"`），仅展开时才触发虚拟化渲染。

### 3.4 方案 C：ItemTemplate 扁平化 + OneTime 绑定（P1）

**优化步骤**：

```xml
<!-- 改造前 -->
<DataTemplate x:DataType="vm:VaccineItem">
    <Border Background="{StaticResource WeuiBg1Brush}" CornerRadius="6" Padding="10,8" Margin="0,4,0,0">
        <Grid ColumnDefinitions="Auto,*,Auto">
            <Border Grid.Column="0" Background="{StaticResource WeuiBg2Brush}" CornerRadius="4" Padding="6,2">
                <TextBlock Text="{Binding Category}" FontSize="10" Foreground="{StaticResource WeuiFg1Brush}"/>
            </Border>
            <TextBlock Grid.Column="1" Text="{Binding Name}" FontSize="13" VerticalAlignment="Center" Margin="8,0,0,0"/>
            <TextBlock Grid.Column="2" Text="{Binding DueText}" FontSize="12" Foreground="{StaticResource WeuiFg2Brush}" VerticalAlignment="Center"/>
        </Grid>
    </Border>
</DataTemplate>

<!-- 改造后：所有绑定改为 OneTime（VaccineItem 是只读 POCO，属性不会变） -->
<DataTemplate x:DataType="vm:VaccineItem">
    <Border Background="{StaticResource WeuiBg1Brush}" CornerRadius="6" Padding="10,8" Margin="0,4,0,0">
        <Grid ColumnDefinitions="Auto,*,Auto">
            <Border Grid.Column="0" Background="{StaticResource WeuiBg2Brush}" CornerRadius="4" Padding="6,2">
                <TextBlock Text="{Binding Category, Mode=OneTime}" FontSize="10" Foreground="{StaticResource WeuiFg1Brush}"/>
            </Border>
            <TextBlock Grid.Column="1" Text="{Binding Name, Mode=OneTime}" FontSize="13" VerticalAlignment="Center" Margin="8,0,0,0"/>
            <TextBlock Grid.Column="2" Text="{Binding DueText, Mode=OneTime}" FontSize="12" Foreground="{StaticResource WeuiFg2Brush}" VerticalAlignment="Center"/>
        </Grid>
    </Border>
</DataTemplate>
```

**预期收益**：`OneTime` 绑定跳过后续变更监听订阅，单项创建耗时 -30%。

### 3.5 方案 D：UseLayoutRounding + 不透明背景（P2）

**优化步骤**：

1. 在 HomeView 根节点加 `UseLayoutRounding="True"`
2. 检查 WeUI 样式中是否有透明背景（`#80xxxxxx`），改为不透明色

### 3.6 方案 E：性能埋点 + 基准测试（P2）

**优化步骤**：

1. 在 `HomeViewModel.ToggleVaccinePanel` 命令前后加 `Stopwatch` 埋点
2. 新增 `VaccineListRenderTests` 基准测试
