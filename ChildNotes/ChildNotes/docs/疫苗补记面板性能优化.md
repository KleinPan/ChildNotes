# 疫苗追踪"补记"面板性能优化文档

## 一、问题背景

用户反馈：在疫苗追踪系统中，点击"补记"按钮后，从页面底部弹出的操作面板存在严重的性能问题，表现为响应延迟明显、界面卡顿。

## 三、技术调研：Avalonia 虚拟化方案

### 3.1 WPF vs Avalonia 虚拟化差异

| 特性 | WPF | Avalonia 12 |
|------|-----|-------------|
| 虚拟化面板 | `VirtualizingStackPanel`（内置） | 已移除，不兼容 |
| 推荐方案 | - | `ItemsRepeater` + `StackLayout`/`UniformGridLayout` |
| `ListBox` 虚拟化 | 默认启用 | 默认启用（但项目未用 `ListBox`） |
| `ItemsControl` 虚拟化 | 需手动配置 | **不支持**（默认 `StackPanel` 非虚拟化） |

### 3.2 本项目适用性评估

- **项目 Avalonia 版本**：12.0.5（见 [Directory.Packages.props](file:///e:/0_Code/5_Git/AiJi/ChildNotes/Directory.Packages.props)）
- **当前列表控件**：全项目使用 `ItemsControl`，零虚拟化
- **ItemsRepeater 可行性**：Avalonia 12 的 `ItemsRepeater` 位于 `Avalonia.Controls` 命名空间，支持 `StackLayout` 虚拟化。但时间轴是嵌套布局（三列：免费/时间/自费），且每个分组高度不固定，直接套用 `ItemsRepeater` 会破坏三列对齐结构。

### 3.3 最终技术选型

鉴于时间轴的嵌套三列布局特性，**纯虚拟化方案不适用**（会破坏列对齐）。采用"组合优化"策略：

| 优化点 | 方案 | 预期收益 |
|--------|------|----------|
| 数据加载 | `Load()` → `LoadAsync()`，DB 读取+构建移至后台线程 | UI 线程不阻塞，抽屉先弹出占位，数据异步填充 |
| 转换器消除 | 7 个 `Classes.xxx` 绑定改为预计算 bool 属性 | 每剂次减少 7 次转换器调用（280 次→0 次） |
| 弹出响应 | 疫苗类型先设 `IsRecordSheetOpen=true` 再 `await OpenAsync` | 抽屉立即弹出，数据加载与弹出并行 |
| 性能可观测 | `Stopwatch` 埋点 + `DevLogger` 输出 | 运行时可测量各阶段耗时 |

### 4.2 VaccineFormViewModel 异步加载

[VaccineFormViewModel.cs:46-81](file:///e:/0_Code/5_Git/AiJi/ChildNotes/ChildNotes/ViewModels/Forms/VaccineFormViewModel.cs#L46)

```csharp
public async Task LoadAsync()
{
    var birthDate = ...;
    var customSnapshot = CustomVaccines.ToList();  // 快照避免并发访问

    // 后台线程：DB 读取 + 计划构建 + 状态计算 + 分组
    var groups = await Task.Run(() =>
    {
        var vaccineRecords = _recordService.GetByType(RecordType.Vaccine, 1000);
        var plans = VaccineTimelineBuilder.BuildPlans(birthDate, customSnapshot);
        var views = VaccineTimelineBuilder.BuildPlanViews(plans, vaccineRecords, birthDate, today);
        return VaccineTimelineBuilder.BuildGroups(views);
    });

    // UI 线程：仅做集合同步
    TimelineGroups.Clear();
    foreach (var g in groups) TimelineGroups.Add(g);
}
```

**⚠️ 关键：避免同步阻塞异步方法**

异步方法（`LoadAsync`/`OpenAsync`/`MarkVaccineDoneAsync` 等）**不可**通过 `.GetAwaiter().GetResult()` 或 `.Wait()` 同步阻塞调用，否则在 UI 线程会引发**死锁**：

- `await Task.Run(...)` 默认捕获 UI 线程的 `SynchronizationContext`
- continuation（集合同步部分）需要回到 UI 线程执行
- 但 UI 线程被 `GetResult()` 阻塞，continuation 永远等不到 → 死锁

正确做法：事件处理器使用 `async void` 直接 `await`（fire-and-forget 模式），详见 [RecordSheetView.axaml.cs:230-244](file:///e:/0_Code/5_Git/AiJi/ChildNotes/ChildNotes/Views/RecordSheetView.axaml.cs#L230)：

```csharp
private async void OnVaccineMarkDone(object sender, RoutedEventArgs e)
{
    if (sender is Button btn && btn.DataContext is VaccinePlanView plan
        && DataContext is RecordSheetViewModel vm)
    {
        await vm.MarkVaccineDoneAsync(plan);  // 直接 await，不阻塞 UI
    }
}
```

## 六、优化前后对比

| 指标 | 优化前 | 优化后 | 改善 |
|------|--------|--------|------|
| UI 线程阻塞 | 同步执行 DB+构建（100-500ms） | 后台线程，UI 线程仅集合同步（<50ms） | 阻塞消除 |
| 弹出响应 | 等数据加载完才显示 | 立即弹出占位 | 响应即时 |
| 转换器调用 | 280 次/次加载（40 剂次 × 7） | 0 次 | 100% 消除 |
| 数据构建 | 58-104ms（单元测试） | 同（未改算法） | 持平 |
| 可观测性 | 无 | 三阶段耗时埋点 | 可量化 |

## 七、后续可扩展优化（未实施）

如需进一步优化，可考虑：

1. **ItemsRepeater 虚拟化**：将时间轴外层 `ItemsControl` 改为 `ItemsRepeater` + `StackLayout`，仅渲染可见分组。需重构三列布局结构。
2. **表单懒加载**：12 个表单 `StackPanel` 改为 `ContentControl` + `DataTemplateSelector` 按需实例化，减少初始可视树节点。
3. **状态增量更新**：`MarkVaccineDone/Skipped` 后不重建整个时间轴，仅更新单个剂次的状态字段。
4. **CompiledBindings**：项目已启用 `AvaloniaUseCompiledBindingsByDefault=true`，但 `Classes.xxx` 绑定可能未走编译路径，可验证。
