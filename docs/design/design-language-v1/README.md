# Design Language v1 指导摘要

本文件保留 ChildNotes 后续设计演进的关键方向。详细方案不再分散成多份长文，新增设计决策应优先沉淀到 `../design-language.md`、`../design-system.md`、`../components.md` 或 `../design-tokens.md`。

## 一句话原则

让家长用最少负担记录宝宝成长，并在需要时获得温和、可信、可解释的 AI 帮助。

## 体验原则

- **记录优先**：核心路径是快速记录、补记、查看时间线。
- **降低压力**：文案避免评判，异常与提醒使用温和语气。
- **内容可信**：AI 只做整理、提示与建议，不替代医疗或育儿判断。
- **跨端一致**：移动端优先，桌面端放大信息密度但不改变任务顺序。
- **可持续扩展**：新增组件、Token、页面模板必须能复用现有规则。

## 视觉与组件

- 卡片承载记录主体，保持清晰层级和足够留白。
- 主操作只保留一个，次要操作收敛到菜单、Sheet 或详情页。
- 状态必须明确：加载、空态、错误、成功、同步中、AI 生成中。
- 图标和插画服务信息表达，不做无意义装饰。

## Token 方向

- 颜色按语义命名：品牌、背景、文本、边框、成功、警告、错误、AI。
- 间距、圆角、阴影按层级递增，不为单个页面创建孤立值。
- 动效只用于反馈状态变化，时长保持短、轻、可预期。

## AI 体验

- 先展示来源与置信边界，再展示结论。
- 提供可编辑结果，避免用户被 AI 输出锁定。
- 高风险内容必须给出免责声明和人工判断入口。

## 维护规则

- 本文件只保留方向，不继续扩写成长篇规范。
- 具体规则进入 `../design-language.md`、`../components.md`、`../design-tokens.md`。
- 如果需要新增大段研究或阶段性材料，请放入 `../../archive/`。
