# Design System v0 指导摘要

本文件保留早期 Design System 中仍有指导价值的内容。后续落地规则以 `../design-system.md` 与 `../design-tokens.md` 为准。

## 核心方向

- **温暖但专业**：面向家庭场景，视觉要亲和、可信、长期可用。
- **内容优先**：记录、时间线、图表和 AI 总结应比装饰更突出。
- **克制统一**：颜色、圆角、阴影、字号和间距尽量复用 Token。
- **可访问**：保证基础对比度、触控尺寸、状态反馈和错误可恢复。

## 品牌与视觉

- 品牌色用于主行动、重点强调和品牌识别，不滥用。
- 背景与卡片使用低干扰色，避免压过宝宝记录内容。
- 成功、警告、错误等语义色只表达状态，不替代说明文案。

## 维护规则

- 本目录不再维护多份长文，新增规则写入上层精简规范。
- 若旧文件内容需要恢复，优先提炼为短规则再加入当前规范。
