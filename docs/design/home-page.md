# 首页设计规范

版本：v1.0
页面：首页（Home）

## 1. 页面定位

首页不是数据看板，而是：**宝宝今日状态中心**。

用户打开首页，需要快速知道：

- 宝宝今天状态怎么样？
- 有没有重要提醒？
- 如何快速记录宝宝发生的事情？

最终体验：打开 APP 3 秒知道——宝宝今天怎么样，以及下一步该做什么。

## 2. 产品原则

### 首页展示「状态」，不是「流水」

**首页负责：**

- ✅ 宝宝状态
- ✅ AI 洞察
- ✅ 重要提醒
- ✅ 快速记录入口

**首页不负责：**

- ❌ 完整记录列表
- ❌ 所有历史数据
- ❌ 详细成长趋势
- ❌ 大量快捷按钮

### 数据归属

| 信息 | 所属页面 |
|---|---|
| 喂奶记录 | 记录 |
| 睡眠记录 | 记录 |
| 尿布记录 | 记录 |
| 辅食记录 | 记录 |
| 身高体重趋势 | 成长 |
| AI 分析 | 首页 |

## 3. 页面结构

```text
Home 首页
│
├── Baby Header
│
├── Today Insight
│
├── Important Reminder
│
├── AI Record Input
│
├── Quick Record Sheet
│
└── Bottom Navigation
```

## 4. Baby Header 宝宝信息区域

**目的**：展示宝宝身份和基础成长信息。**不是功能入口**。

**内容**：宝宝头像、宝宝昵称、当前年龄、身高、体重、少量操作入口。

**示例**：

```
┌────────────────────────┐
│  👶 小铃铛              │
│  8个月28天              │
│  身高 68cm    体重 7.6kg│
│                        │
│                       ✓│
└────────────────────────┘
```

**设计规则**：

| 项 | 要求 |
|---|---|
| 高度 | 90 ~ 110dp |
| 布局 | 信息紧凑、留白充足、不使用大面积卡片 |
| 禁止 | 身高独立卡片 / 体重独立卡片 / 成长趋势图放首页 |

**原因**：身高体重属于成长档案，不属于每日决策信息。

## 5. Today Insight 今日状态

**定位**：首页核心模块。展示 **AI 理解后的宝宝状态**，不是 AI 聊天机器人。

**AI 负责**：总结宝宝当天情况、发现变化、提供建议。

**示例**：

```
😊 小铃铛今天状态不错

吃奶规律
晚上睡眠比昨天增加 40 分钟
继续保持当前作息

              查看详情 >
```

**禁止**：不要展示「今日数据」如下，因为属于记录数据，会和记录页面重复：

```
今日数据
吃奶 6 次
睡眠 10 小时
尿布 4 次
```

## 6. Important Reminder 重要提醒

**用途**：展示低频高价值信息，例如疫苗、体检、异常情况、成长提醒。

**示例**：

```
💉 疫苗提醒

A 群流脑多糖疫苗 第 2 剂

还有 2 天

              查看 >
```

**规则**：

- 有提醒 → 显示
- 无提醒 → 隐藏
- 不要长期固定占据首页空间

## 7. AI Record Input AI 记录入口

**产品核心入口**。用户最自然的行为是：一句话告诉 APP 发生了什么。

**UI**：

```
┌────────────────────────┐
│ 记录宝宝今天发生的事... │
│                        │
│                        │
│                       +│
│ AI 自动识别分类         │
│ 今日剩余 8/10 次        │
└────────────────────────┘
```

**额度规则**：

| 用户类型 | 能力 | 每日次数 |
|---|---|---|
| 免费用户 | AI 记录 | 每日 10 次 |
| 会员用户 | AI 成长助手 | 每日 100 次 |

**设计目标**：让用户感觉——我是在记录宝宝，不是在调用 AI。

## 8. Quick Record 快速记录

**定位**：AI 记录的降级方案。

**使用场景**：AI 次数不足、用户想快速记录、网络异常。

**默认状态**：隐藏。点击 `+` 展开 Bottom Sheet。

**Bottom Sheet 内容**：

```
快速记录

🍼 喂奶
😴 睡眠
💩 换尿布
🌡 体温
💊 用药
🥣 辅食
📏 成长
```

**设计规则**：不要首页常驻九宫格。首页应该保持干净。

## 9. Bottom Navigation

**一级导航推荐**：首页 / 记录 / 成长 / 我的。

**不推荐**：首页 / 喂养 / 成长 / 我的。

**原因**：「喂养」范围太窄，实际包含喂奶、睡眠、尿布、辅食、用药，「记录」更符合产品定位。

## 10. 信息权重

首页视觉优先级：

| 区域 | 优先级 |
|---|---|
| 宝宝信息 | ★★★★★ |
| 今日状态 | ★★★★★ |
| 提醒 | ★★★☆☆ |
| AI 记录入口 | ★★★☆☆ |
| 快速记录 | ★☆☆☆☆ |

## 11. 开发约束（AI Coding Rules）

AI 开发首页时必须遵守：

**禁止**：

- 增加重复数据模块
- 增加大面积统计卡
- 增加首页时间轴
- 默认展开快捷按钮
- 添加无意义装饰

**必须**：

- 使用 Design Tokens
- 使用统一组件
- 保持信息层级
- 优先复用已有组件

## 12. 最终设计目标

宝宝日记首页应该像：

```
微信的简单易用
+
Notion 的信息结构
+
宝宝成长 App 的温暖感
+
AI 助手的智能感
```

最终体验：打开 APP 3 秒知道——宝宝今天怎么样，以及下一步该做什么。
