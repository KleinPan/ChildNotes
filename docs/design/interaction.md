# 交互规范

版本：v1.0

## 1. 交互设计目标

宝宝日记是一款高频、低压力的宝宝成长记录应用。

**交互目标**：

- 快速记录
- 减少操作步骤
- 降低用户负担
- 让记录像聊天一样自然

**核心原则**：

> 用户记录宝宝，而不是操作一个复杂系统。

## 2. 交互原则

### 2.1 少打断

默认不要弹窗。优先使用：页面内反馈、Bottom Sheet、Toast。

禁止频繁使用：确认弹窗、提示弹窗、强制引导。

### 2.2 渐进展示

复杂功能默认隐藏，用户需要时展开。

例如快速记录：默认只显示 `+`，点击后展开喂奶 / 睡眠 / 尿布 / 辅食等。

### 2.3 即时反馈

所有用户操作必须有反馈。

例如点击记录：立即按钮状态变化 + Toast + 数据更新。避免点击后没有反应。

## 3. 页面跳转规则

| 层级 | 方式 | 示例 |
|---|---|---|
| 一级页面 | Bottom Navigation | 首页 / 记录 / 成长 / 我的 |
| 二级页面 | 页面进入 | 首页 → 查看今日状态 → 详情页 |
| 三级操作 | Bottom Sheet | 首页 → `+` → 快速记录 |

## 4. Bottom Sheet 规范

**使用场景**：

- 适合：新增记录、快捷操作、选择类型
- 不适合：展示大量内容、长流程编辑

**动画**：从底部滑入，200-300ms。

**样式**：

| 项 | 规范 |
|---|---|
| 顶部圆角 | 24dp |
| 顶部 | 拖拽条 |

## 5. + 按钮交互规范

**默认状态**：首页只显示 `+`，不要展开九宫格。

**点击后**：显示 Quick Record Sheet，内容：🍼 喂奶 / 😴 睡眠 / 💩 尿布 / 🌡 体温 / 💊 用药 / 🥣 辅食 / 📏 成长。

**关闭方式**：

- 点击遮罩关闭
- 下滑关闭
- 返回键关闭

## 6. AI 记录交互

**产品定位**：AI 记录是主要记录方式，不是聊天功能。

### 输入流程

```
用户输入：宝宝下午喝了 120ml 奶

系统 AI 识别：
  类型：喂奶
  时间：下午
  数量：120ml

用户确认：保存
```

### AI 识别失败

不要报错。应允许用户修改。

示例：

```
我没有理解，可以选择记录类型：
🍼 喂奶  😴 睡眠  💩 尿布
```

## 7. AI 次数限制交互

| 状态 | 条件 | 交互 |
|---|---|---|
| 正常 | 剩余 > 2 次 | 显示"今日剩余 8/10 次" |
| 接近限制 | 剩余 ≤ 2 次 | 提示"今日 AI 记录剩余 2 次，升级会员获得 100 次每日额度" |
| 用完 | 剩余 0 次 | **不阻止记录**，提供快速记录入口："AI 次数已用完，可以继续使用快速记录" |

## 8. Record Timeline 交互

**时间轴定位**：记录页面展示所有宝宝事件。

示例：

```
09:30  🍼 喂奶    120ml
11:20  😴 睡眠    1 小时
```

### 筛选

支持标签筛选：全部 / 喂奶 / 睡眠 / 尿布 / 辅食 / 用药。

### 编辑

点击记录进入详情，支持修改和删除。

### 删除

不要直接删除。推荐轻提示确认："删除这条记录？"

## 9. 手势规范

| 手势 | 用途 | 规则 |
|---|---|---|
| 上下滑动 | 列表滚动 | 支持 |
| 左滑 | 记录操作（如删除） | **仅用于记录**，不要所有地方都支持左滑 |
| 长按 | 高级操作 | 谨慎使用，只用于高级操作 |

## 10. Loading 状态

| 场景 | 规范 |
|---|---|
| 数据加载 | 优先骨架屏 |
| AI 生成 | 显示"AI 正在整理宝宝记录..."，不要复杂动画 |

## 11. Empty State

无数据时不要显示空白。

示例：

```
今天还没有记录
点击 + 开始记录宝宝成长
```

## 12. Error Handling

### 网络错误

| 项 | 内容 |
|---|---|
| 禁止 | 技术错误（如 `Network Error`） |
| 应该 | "暂时无法连接，请稍后重试" |

### AI 错误

| 项 | 内容 |
|---|---|
| 禁止 | 暴露模型错误（如 `API timeout`） |
| 应该 | "AI 暂时没有完成分析，可以稍后再试" |

## 13. Animation Rules

**推荐动画**：时间 200-300ms，用于页面切换、Bottom Sheet、状态变化。

**禁止**：大量粒子动画、自动播放动画、影响记录效率的动画。

## 14. Accessibility

交互元素必须：

- 可点击区域 ≥ 40dp
- 文字清晰
- 颜色不能作为唯一提示

## 15. AI Development Rules

AI 修改交互时的开发规则统一见 [`components.md`](components.md) 的"AI 开发规则"章节（含 AI 生成 UI、AI 新增组件、AI 修改交互三组规则）。

## 16. 最终交互目标

宝宝日记交互体验应该接近：

- 微信的自然记录
- Notion 的信息组织
- 宝宝成长助手的温暖感

最终：用户打开 APP——3 秒知道宝宝状态，1 步完成记录，不需要学习。
