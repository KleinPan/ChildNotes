# ChildNotes Design Language

## Brand & Experience System Specification

版本：v0.1

---

# 1. Brand Experience Overview

ChildNotes 不只是一个记录工具，而是一份持续成长的家庭记忆档案。

品牌体验目标：

> 让父母多年以后打开 ChildNotes，依然能感受到孩子成长时的温度。

---

# 2. Emotional Design Principles

## Warm 温暖

表现：

- 柔和视觉
- 自然语言
- 有温度的反馈

避免：

- 冷冰冰的数据面板
- 过度专业术语

---

## Precious 珍贵

每一次记录都应该被认真对待。

例如：

普通：

> 创建成功

ChildNotes：

> 已保存这个珍贵瞬间

---

## Growth 成长

产品应该体现时间流动。

核心表达：

```
今天
 ↓
这个月
 ↓
这一年
 ↓
整个成长旅程
```

---

# 3. Voice & Tone

## 产品语言

应该：

- 亲切
- 简洁
- 鼓励

避免：

- 命令式
- 机械式
- 焦虑式

---

# 4. Memory Experience

记录不是事件列表。

应该形成：

```
Moment
  ↓
Memory
  ↓
Story
```

AI 可以帮助把碎片记录连接成成长故事。

---

# 5. Growth Timeline Experience

时间轴是核心体验。

设计方向：

- 连续
- 有生命感
- 支持回忆

不是：

```
数据库列表
```

而是：

```
成长旅程
```

---

# 6. Trust Experience

儿童数据具有长期价值。

体验要求：

- 数据透明
- 用户掌控
- AI边界清晰

---

# 7. Experience Checklist

- [ ] 用户感受到陪伴
- [ ] 记录过程无压力
- [ ] 回忆体验有价值
- [ ] AI增强而非替代
