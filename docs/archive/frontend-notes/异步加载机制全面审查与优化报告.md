## 一、审查结论速览

### 2.2 优化方案

#### 方案 A：HomeViewModel 异步化 + 合并查询（优先级 P0）

**问题**：[HomeViewModel.Refresh()](file:///e:/0_Code/5_Git/AiJi/ChildNotes/ChildNotes/ViewModels/HomeViewModel.cs#L90) 是项目内最大的 UI 阻塞点，串行 8+ 次 SQLite 查询，每次切换 Tab 回 Home、保存记录、改宝宝都会触发。

**优化步骤**：

1. **合并当天记录查询**：`GetDayStats(Today)` 内部已查 `GetByDate(Today)`，`RefreshLastFeed` 又查一次。改为一次性 `GetByDate(Today)` 在内存中分类聚合，复用给 Stats/Feed/Diaper/Sleep。

2. **异步化**：参照 [VaccineFormViewModel.LoadAsync()](file:///e:/0_Code/5_Git/AiJi/ChildNotes/ChildNotes/ViewModels/Forms/VaccineFormViewModel.cs#L47) 的 `await Task.Run(...)` 模式：

```csharp
public async Task RefreshAsync()
{
    // 后台线程：一次性查当天全部记录 + 各类型最新记录
    var snapshot = await Task.Run(() =>
    {
        var todayRecords = _recordService.GetByDate(DateTime.Today);
        var latestFeed = _recordService.GetLatest(RecordType.Feed);
        var vaccines = _recordService.GetByType(RecordType.Vaccine, 100);
        var activities = _recordService.GetByType(RecordType.Activity, 100);
        var growth = _recordService.GetByType(RecordType.Growth, 1);
        var abnormal = _recordService.GetByType(RecordType.Abnormal, 1);
        var stats = _statsService.GetDayStats(todayRecords);  // 支持传入已查记录
        return new { todayRecords, latestFeed, vaccines, activities, growth, abnormal, stats };
    });

    // UI 线程：仅做属性赋值和集合更新
    TodayStats = snapshot.stats;
    // ... 从 snapshot 中取数据更新各属性
}
```

**预期收益**：
- UI 线程阻塞从 200-500ms 降至 < 50ms（仅属性赋值）
- 当天记录从 3 次查询降至 1 次

#### 方案 C：LoginViewModel 异步化（优先级 P1）

**问题**：[LoginViewModel.Submit()](file:///e:/0_Code/5_Git/AiJi/ChildNotes/ChildNotes/ViewModels/LoginViewModel.cs#L27) 是同步方法，PBKDF2 10000 次迭代同步执行 30-80ms，加上 DB 操作共阻塞 100-200ms。

**优化步骤**：

```csharp
[RelayCommand]
private async Task SubmitAsync()
{
    IsLoading = true;
    try
    {
        // PBKDF2 是 CPU 密集型，必须卸载到线程池
        var result = await Task.Run(() => IsRegisterMode
            ? _auth.Register(Username, Password, NickName)
            : _auth.Login(Username, Password));

        if (result.Success)
        {
            SaveCredentialsToSyncConfig();
            ServiceProvider.Instance.BindUserToState();
            await Task.Run(() => ServiceProvider.Instance.BabyService.LoadBabyList());
            App.RaiseLoginSucceeded();
            LoginSucceeded?.Invoke();
        }
        else
        {
            ErrorMessage = result.Message;
        }
    }
    catch (Exception ex) { /* ... */ }
    finally { IsLoading = false; }
}
```

**预期收益**：登录按钮点击后 UI 立即响应（显示 loading），不再冻结

## 四、API 数据请求与缓存优化

### 4.1 现状：零缓存、零预加载

| 机制 | 现状 | 问题 |
|------|------|------|
| HTTP 缓存 | 无 | 所有 HttpClient 裸 `new`，无 `CacheControl` 头 |

#### 方案 E：AI 请求增加 CancellationToken（优先级 P0，安全关键）

**问题**：[LlmClient.ChatAsync](file:///e:/0_Code/5_Git/AiJi/ChildNotes/ChildNotes/Services/LlmClient.cs#L13) 和 [AiParseApiClient.ParseAsync](file:///e:/0_Code/5_Git/AiJi/ChildNotes/ChildNotes/Services/AiParseApiClient.cs#L18) 都无 CancellationToken，AI 分析最长阻塞 120 秒，期间用户无法取消，UI 显示"生成中"但无法中断。

**优化步骤**：

```csharp
// LlmClient.cs
public async Task<string> ChatAsync(LlmConfig config, string systemPrompt,
    string userContent, CancellationToken ct = default)
{
    // ...
    using var resp = await SendCoreAsync(url, config.ApiKey, body, ct);
    // ...
}

private static async Task<HttpResponseMessage> SendCoreAsync(
    string url, string apiKey, object body, CancellationToken ct)
{
    using var req = new HttpRequestMessage(HttpMethod.Post, url);
    req.Content = new StringContent(JsonSerializer.Serialize(body), Encoding.UTF8, "application/json");
    req.Headers.Authorization = new AuthenticationHeaderValue("Bearer", apiKey);
    return await HttpClient.SendAsync(req, ct);  // 传入 ct
}
```

调用方（`AiAnalysisService.GenerateAsync`、`AiNoteViewModel.Save`）需把自身的 CancellationToken 透传下去。

**预期收益**：用户可随时取消长时间运行的 AI 请求，避免 UI 长时间冻结

#### 方案 F：FamilyApiClient 内存缓存 + 请求去重（优先级 P1）

**问题**：[FamilyViewModel.LoadAsync](file:///e:/0_Code/5_Git/AiJi/ChildNotes/ChildNotes/ViewModels/FamilyViewModel.cs#L48) 每次进页面都全量拉取，`ConfirmJoin`/`ConfirmRole` 后又拉一次。

**优化步骤**：

```csharp
public sealed class FamilyApiClient
{
    private List<BabyFamilyItem>? _cachedFamilies;
    private Task<List<BabyFamilyItem>?>? _inflightTask;
    private DateTime _cacheTime = DateTime.MinValue;
    private static readonly TimeSpan CacheTtl = TimeSpan.FromMinutes(2);

    public async Task<List<BabyFamilyItem>?> ListFamiliesAsync(CancellationToken ct = default)
    {
        // 请求去重：已有进行中的请求则复用
        if (_inflightTask is not null)
            return await _inflightTask;

        // 缓存命中
        if (_cachedFamilies is not null && DateTime.UtcNow - _cacheTime < CacheTtl)
            return _cachedFamilies;

        _inflightTask = DoListFamiliesAsync(ct);
        try
        {
            var result = await _inflightTask;
            if (result is not null)
            {
                _cachedFamilies = result;
                _cacheTime = DateTime.UtcNow;
            }
            return result;
        }
        finally { _inflightTask = null; }
    }

    public void InvalidateCache() => _cachedFamilies = null;
}
```

`ConfirmJoin`/`ConfirmRole` 后调用 `InvalidateCache()` 再 `LoadAsync()`，避免无效缓存。

**预期收益**：2 分钟内重复进入家人页零网络请求；并发请求合并为 1 次

#### 方案 J：启动流程异步化 + 性能埋点（优先级 P0）

**问题**：[App.OnFrameworkInitializationCompleted](file:///e:/0_Code/5_Git/AiJi/ChildNotes/ChildNotes/App.axaml.cs#L30) 全程在 UI 线程同步执行，ServiceProvider 静态构造触发 DbInitializer（13 表 + 13 列迁移）阻塞 UI。

**优化步骤**：

```csharp
// App.axaml.cs
public override void OnFrameworkInitializationCompleted()
{
    var sw = Stopwatch.StartNew();
    DevLogger.Log("Startup", "OnFrameworkInitializationCompleted start");

    // 1. 异常处理器（同步，极快）
    RegisterExceptionHandlers();
    DevLogger.Log("Startup", $"ExceptionHandlers: {sw.ElapsedMilliseconds}ms");

    // 2. SQLite 原生库（同步，必须）
    Batteries_V2.Init();
    DevLogger.Log("Startup", $"Batteries_V2.Init: {sw.ElapsedMilliseconds}ms");

    // 3. 会话恢复 + DB 初始化异步化
    _ = Task.Run(() =>
    {
        TryRestoreSession();        // 触发 ServiceProvider 静态构造 + DbInitializer
        Dispatcher.UIThread.Post(() =>
        {
            DevLogger.Log("Startup", $"Session+DB: {sw.ElapsedMilliseconds}ms");
            ShowInitialWindow();
            DevLogger.Log("Startup", $"Window shown: {sw.ElapsedMilliseconds}ms");
        });
    });

    // 4. 先显示 splash/loading 窗口
    ShowSplashWindow();

    base.OnFrameworkInitializationCompleted();
}
```

**注意**：ServiceProvider 改为非静态构造有较大重构量，可先保持静态构造但把触发时机移到后台线程。

**预期收益**：冷启动白屏时间从 500ms+ 降至 < 100ms（先显示 splash）

```csharp
    conn.Open();
    using var cmd = conn.CreateCommand();
    cmd.CommandText = "PRAGMA busy_timeout=5000; PRAGMA foreign_keys=ON;";
    cmd.ExecuteNonQuery();
    return conn;
}
```

同时移除构造函数中重复的 `journal_mode=WAL`（持久化属性，只需设一次）。

#### 方案 M：启动性能埋点（优先级 P1）

**问题**：启动链路无 Stopwatch 埋点，无法量化各阶段耗时。

**优化步骤**：在 `App.OnFrameworkInitializationCompleted` 入口 `Stopwatch.StartNew()`，在 `Batteries_V2.Init` / `TryRestoreSession` / `EnterMainShell` / `Window shown` 各节点记录 `sw.ElapsedMilliseconds` 并通过 `DevLogger.Log("Startup", ...)` 输出。

**预期收益**：可量化各阶段耗时，为后续优化提供数据支撑
